## 1. 이게 뭔가요? (1분 이해)

- **NotebookLM** = 구글이 만든 "AI 노트". 내가 올린 자료(문서·링크)를 읽고, **요약**해 주고, **질문에 답**해 줍니다.
- 원래는 인터넷 사이트에 들어가 마우스로 씁니다.
- **이 키트(NotebookLM-CLI One-Click Kit)**는 그 NotebookLM 을 **내 컴퓨터 명령**과 **AI 비서(Claude·Cursor)** 에서 쓸 수 있게 해 줍니다.
- 어려운 명령어? **몰라도 됩니다.** 검은 창에서 **번호만 누르면** 됩니다.

> 한마디로: "NotebookLM 을 내 컴퓨터/AI 비서와 연결해 주는 **자동 설치·연결 도우미**" 입니다.

## 2. 준비물 챙기기

설치 전에 아래 **3가지**가 있어야 합니다.

1. **인터넷 연결** — 와이파이나 랜선. (프로그램을 인터넷에서 받아야 합니다.)
2. **구글 계정** — NotebookLM 은 구글 서비스입니다. **평소 쓰는 G메일 계정이면** 됩니다.
   - 없다면: 구글에서 무료로 만들 수 있습니다 (https://accounts.google.com).
3. **크롬(Chrome) 브라우저** — 로그인 창을 띄우는 데 씁니다.
   - 없다면: https://www.google.com/chrome/ 에서 받으세요.

> 이 3가지만 있으면 됩니다. 나머지(파이썬 등)는 설치 과정에서 **자동**으로 처리됩니다.

## 3. 필요 프로그램 다운로드

이 키트가 돌아가려면 **파이썬(Python)** 이라는 기반 프로그램이 필요합니다.
**대부분 설치 과정에서 자동 안내**되지만, 직접 받는 법도 알려드립니다.

### 파이썬 받는 법 (없을 때만)

1. 인터넷 주소창에 **https://www.python.org/downloads/** 입력 → Enter
2. 노란색 **"Download Python 3.x.x"** 버튼 클릭 (이 키트는 **3.10 ~ 3.13** 버전과 맞습니다)
3. 받은 파일을 더블클릭해서 설치 시작
4. ⚠️ **가장 중요**: 첫 화면 맨 아래 **"Add Python to PATH"** 네모칸을 **반드시 체크(✔)** 하세요.
   - 이게 꺼져 있으면 컴퓨터가 파이썬을 못 찾아서 설치가 실패합니다.
5. **"Install Now"** 클릭 → 끝날 때까지 기다리기 → **"Close"**
6. 그다음 이 키트의 `시작하기.bat` 으로 돌아오세요.

> `uv`, `notebooklm-mcp-cli` 같은 나머지 프로그램은 **설치 과정에서 자동으로** 받아집니다.
> 신경 쓰지 않아도 됩니다. (uv 를 먼저 시도하고, 안 되면 pip 로 자동 전환합니다.)

## 4. 설치하기 (그림처럼 따라하기)

1. 키트 폴더를 엽니다. (압축으로 받았다면 **압축을 먼저 풉니다** — 오른쪽 클릭 → "압축 풀기")
2. **`시작하기.bat`** 파일을 찾아 **마우스로 두 번 빠르게 클릭(더블클릭)** 합니다.
3. 검은 창이 열리고 메뉴가 나옵니다. 키보드로 **`1`** 을 누르고 **Enter**.
4. "설치 준비" 안내가 나오면, **아무 키나** 눌러 진행합니다.
5. 설치가 **6단계로 자동** 진행됩니다 (사람이 할 일 없음, 기다리기만):

   | 단계 | 화면 표시 | 하는 일 |
   |---|---|---|
   | `[1/6]` | 인터넷 연결 확인 | 인터넷이 되는지 점검 |
   | `[2/6]` | 파이썬 찾기 | 컴퓨터에서 파이썬을 자동 탐색 |
   | `[3/6]` | pip 확인 | 설치 도우미(pip) 준비 |
   | `[4/6]` | uv 확인/설치 | 빠른 설치 도구(uv) 준비 |
   | `[5/6]` | notebooklm-mcp-cli 설치 | **인터넷으로 본체 받기 (조금 기다림)** |
   | `[6/6]` | 설치 확인 | 잘 깔렸는지 `nlm` 버전 확인 |

6. **"[설치 완료] / INSTALLATION COMPLETE"** 글자가 보이면 성공입니다. 아무 키나 눌러 닫습니다.

> ⚠️ 중간에 **"파이썬을 찾을 수 없습니다"** 가 나오면 → 위 [3장](#3-필요-프로그램-다운로드)대로 파이썬을 먼저 받고
> (꼭 **"Add Python to PATH" 체크**), 다시 `[1] 설치하기` 를 누르세요.

## 5. 실행·사용하기

1. `시작하기.bat` 더블클릭 → **`2`**(사용하기) → Enter
2. "사용 메뉴(RUN)"가 열립니다.
3. **처음이라면 `S`(쉬운 시작)** 를 누르세요. 아래를 **자동으로 차근차근** 안내합니다:
   - **1단계 로그인 확인/로그인**: 크롬 창이 뜨면 평소처럼 구글 계정으로 로그인
   - **2단계 AI 연결**(선택): Claude Code(`1`) 또는 Cursor(`2`) 선택 (나중에 해도 됨 → Enter)
   - **3단계 첫 노트북 만들기**(선택): 이름 입력 (건너뛰려면 그냥 Enter)
4. 준비 끝! 이제 `[9] 노트북 목록`, `[1] 서버 시작` 등을 자유롭게 씁니다.

> 💡 화면 맨 위에 **"[!] 아직 설치가 안 됐습니다"** 가 보이면, 먼저 `[1] 설치하기` 부터 하세요.

| **T** | 용어 풀이 | MCP·pip 등 어려운 단어를 쉬운 말로 보여줍니다 |

| 3 | 로그인 확인 | 지금 로그인돼 있는지 점검 |

| 8 | nlm 직접 명령 | `nlm` 뒤 명령을 직접 입력 (고급). 예: `notebook list` (그냥 Enter 치면 도움말) |

| 14 | 강제 최신 업데이트 | 최신 버전을 **강제로 다시 설치** (버전이 옛날일 때) |

| 16 | 환경 정보 | 설치 위치, 파이썬 위치, 인증 폴더 등 확인 |
| 0 | 종료 | 메뉴 닫기 |

## 7. AI 비서에 연결하기

Claude Code 나 Cursor 에서 NotebookLM 을 쓰고 싶다면:

1. 사용 메뉴에서 **`15`** → Enter
2. 연결 방법을 고릅니다:
   - **`1`** Claude Code 자동 연결
   - **`2`** Cursor 자동 연결
   - **`3`** 지원 도구 JSON 자동 연결
   - **`4`** 수동 안내 (Claude Desktop / VS Code / Gemini 용 JSON 예시 보기)
3. "연결을 시도했습니다" 가 나오면 성공입니다.
4. **AI 비서 프로그램을 완전히 껐다가 다시 켜세요.** (그래야 연결이 적용됩니다.)

> 설정 파일을 **직접 건드릴 필요가 없습니다.** 자동으로 됩니다.
> 직접 설정하려면 `[15] → 4`(수동 안내)에서 JSON 예시와 파일 위치를 볼 수 있습니다.

**수동 설정용 JSON** (자동이 안 될 때만):

```json
{
  "mcpServers": {
    "notebooklm-mcp": { "command": "notebooklm-mcp" }
  }
}
```

설정 파일 위치: Claude Desktop `%APPDATA%\Claude\claude_desktop_config.json` ·
Cursor `%USERPROFILE%\.cursor\mcp.json` · VS Code `%USERPROFILE%\.vscode\mcp.json` ·
Gemini CLI `%USERPROFILE%\.gemini\settings.json`

## 8. 작동 방식 (워크플로우)

```
1) 설치    : 시작하기.bat → [1]      (notebooklm-mcp-cli 자동 다운로드·설치)
2) 로그인  : [2] 사용 → [2] 로그인     (크롬 → 구글 로그인, 인증정보는 내 PC에만 저장)
3) 연결    : [15] → Claude Code/Cursor (선택 — 안 해도 됨)
4) 사용    : [1] 서버 시작 / [9] 노트북 목록 / AI 비서가 자동 호출
```

- **처음 한 번만** 설치·로그인하면 됩니다.
- 그다음부터는 `[2] 사용하기`로 바로 들어가 씁니다.
- 인증 정보는 **내 컴퓨터(`%USERPROFILE%\.notebooklm-mcp-cli`)에만** 저장됩니다.

## 9. 문제·오류 대처 (풀버전)

> 💡 **막히면 일단 `[5] 진단하기`(`nlm doctor`).** 대부분 원인을 알려줍니다.

### 9-1. 창이 켜자마자 바로 닫혀요 (가장 흔함)

원인은 보통 (a) `.bat` 파일들이 흩어져 있거나 (b) 압축을 안 풀었거나 입니다.

해결:
1. `시작하기.bat`, `INSTALL.bat`, `RUN.bat`, `UNINSTALL.bat` 이 **같은 폴더**에 함께 있는지 확인.
2. 압축(zip)으로 받았다면 **압축을 풀고** 푼 폴더에서 실행.
3. 그래도 닫히면, 글자를 남겨 원인 보기:
   - 키보드 **`Win`키** → `cmd` 입력 → Enter (검은 창이 열림)
   - 파일 탐색기에서 키트 폴더의 **주소창 경로를 복사**
   - 검은 창에 `cd ` (한 칸 띄고) + 붙여넣기 → Enter
   - `시작하기.bat` 입력 → Enter
   - 이제 오류 글자가 화면에 **남습니다.** 그 내용을 보고 아래 표에서 찾으세요.

### 9-2. 증상별 해결표

| 증상 | 해결 |
|---|---|
| `nlm` 명령이 없다고 나옴 | 아직 설치 전 → `[1] 설치하기` |
| "Python 을 찾을 수 없습니다" | [3장](#3-필요-프로그램-다운로드)대로 파이썬 설치 (**PATH 체크!**) |
| 설치는 됐는데 RUN 에서 안 잡힘 | 창 닫고 `시작하기.bat` 다시 더블클릭 → `[2]` (PATH 갱신) |
| 인터넷 연결 경고가 뜸 | 와이파이/랜선·방화벽 확인 후 다시 시도 |
| 로그인 창이 안 뜸 | 크롬 설치 확인 → `[7] 로그아웃` → `[2] 재로그인` |
| 자동 로그인이 계속 실패 | `[4] 수동 로그인` (쿠키 파일 사용) |
| 버전이 오래된 듯 | `[14] 강제 최신 업데이트` |
| AI 비서에 연결했는데 안 보임 | **AI 비서 프로그램을 완전히 껐다 켜기** |
| 원인 불명 | **`[5] 진단하기`** → 결과 안내 따라가기 |

## 10. 자주 묻는 질문 (FAQ)

**Q. 돈이 드나요?**
A. 이 키트와 원본 도구는 **무료**입니다. NotebookLM 자체는 구글 정책을 따릅니다(무료 범위 있음).

**Q. 인터넷이 꼭 필요한가요?**
A. **설치할 때**와 **NotebookLM 을 쓸 때** 필요합니다.

**Q. 제 자료가 외부로 새나요?**
A. 구글 로그인 정보는 **내 컴퓨터에만** 저장됩니다. 이 키트는 자료를 따로 어디에 보내지 않습니다.

**Q. 맥(Mac)에서도 되나요?**
A. 이 키트의 `.bat` 파일은 **윈도우 전용**입니다. 맥은 원본 도구(`notebooklm-mcp-cli`)를 직접 설치해 쓰세요.

**Q. 설치를 잘못한 것 같아요. 처음부터 다시 하려면?**
A. `[4] 제거하기` 로 깨끗이 지운 뒤 `[1] 설치하기` 를 다시 하세요.

**Q. 파이썬을 지워도 되나요?**
A. 이 키트는 파이썬을 **지우지 않습니다.** 다른 프로그램이 쓸 수 있으니 그대로 두길 권합니다.

**Q. 계정이 여러 개예요. 구분해서 쓸 수 있나요?**
A. 네. `[2] 로그인` 때 **프로필 이름**을 입력하면 계정별로 따로 저장됩니다. `[6] 프로필 목록`에서 확인합니다.

## 11. 제거하기

1. `시작하기.bat` → **`4`**(제거하기) → Enter
2. 정말 지울 거면 **`yes`** 입력 → Enter
3. **6단계로 자동 정리**됩니다:
   - AI 도구 연결 해제 → uv 로 제거 → pip 로 제거 → 로그인 데이터 삭제 → 최종 확인
4. **"[제거 완료] / UNINSTALL COMPLETE"** 가 나오면 끝.
5. **파이썬은 그대로 남습니다**(의도된 동작 — 다른 프로그램이 쓸 수 있으므로).

> 혹시 AI 비서 연결이 자동으로 안 지워지면, [13장](#13-파일--문서--데이터-위치)의 설정 파일에서
> `notebooklm-mcp` 부분만 직접 지우면 됩니다.

## 12. 용어 풀이

| 용어 | 쉬운 설명 |
|---|---|
| **NotebookLM** | 내 자료를 읽고 요약·답변해 주는 구글의 AI 노트 |
| **nlm** | NotebookLM 을 컴퓨터 명령으로 다루는 도구 이름 |
| **MCP** | AI 비서(Claude·Cursor)와 이 도구를 잇는 "다리". 켜면 AI 가 내 NotebookLM 을 봅니다 |
| **서버** | 메뉴 `[1]` 로 켜는 "대화 창구". 끄려면 `Ctrl+C` |
| **Python / pip** | 이 도구가 돌아가는 데 필요한 기반 프로그램·설치 도우미 (자동 처리) |
| **uv** | pip 보다 빠른 최신 설치 도구 (자동) |
| **PATH** | 컴퓨터가 도구(`nlm`)를 찾는 길. 설치 후 **새 창에서** 인식됩니다 |
| **프로필** | 계정마다 따로 저장하는 로그인 이름표 |
| **쿠키(cookie)** | 로그인 상태를 담은 작은 파일. 수동 로그인 때 사용 |

## 14. 명령어 모음 (참고용)

> 메뉴를 쓰면 직접 입력할 필요는 **없습니다.** RUN 메뉴 `[8]`에서 `nlm` **뒤의 부분만** 입력하면 됩니다.

| 명령어 | 설명 |
|---|---|
| `nlm login` | 구글 로그인 (크롬 자동) |
| `nlm login --check` | 로그인 상태 확인 |
| `nlm notebook list` | 내 노트북 목록 |
| `nlm notebook create "이름"` | 새 노트북 만들기 |
| `nlm notebook query` | 노트북에 질문하기 |
| `nlm source add` | 노트북에 자료(URL·텍스트·드라이브·파일) 추가 |
| `nlm studio create` | 오디오·영상 등 콘텐츠 생성 |
| `nlm download <종류>` | 만든 결과물 내려받기 |
| `nlm setup add claude-code` | Claude Code 에 자동 연결 |
| `nlm setup add cursor` | Cursor 에 자동 연결 |
| `nlm setup list` | 연결된 AI 도구 보기 |
| `nlm setup remove <도구>` | AI 도구 연결 해제 |
| `nlm doctor` | 문제 자동 진단 |
| `nlm --ai` | AI 모드 (AI 비서용 도움말) |
| `nlm --version` | 버전 확인 |
| `notebooklm-mcp` | MCP 서버 실행 (AI 비서용) |
